# Enterprise Canonical Data Model

To prevent microservices from developing siloed dialects, Aivana enforces an Enterprise Canonical Data Model. Every service must consume and emit payloads conforming to these universal schemas. This acts as Aivana's internal "FHIR for Insurance."

---

## 1. Patient (`Aivana.Patient`)
Represents the insured individual.
```json
{
  "patientId": "UUID",
  "identifiers": {
    "abdm_id": "string",
    "hospital_mrn": "string"
  },
  "demographics": {
    "firstName": "string",
    "lastName": "string",
    "dateOfBirth": "YYYY-MM-DD",
    "gender": "enum[M, F, O]"
  },
  "contact": { "phone": "string", "email": "string" }
}
```

## 2. Policy (`Aivana.Policy`)
Represents the financial contract between Patient and Insurer.
```json
{
  "policyId": "UUID",
  "insurerId": "UUID",
  "policyNumber": "string",
  "productName": "string",
  "coverageDetails": {
    "sumInsuredAmount": "number",
    "roomRentCapType": "enum[PERCENTAGE, FLAT_RATE, NO_CAP]",
    "roomRentCapValue": "number"
  },
  "status": "enum[ACTIVE, EXPIRED, CANCELLED]"
}
```

## 3. Hospital (`Aivana.Hospital`)
Represents the provider organization.
```json
{
  "hospitalId": "UUID",
  "rohiniId": "string",
  "name": "string",
  "tier": "enum[PLATINUM, GOLD, SILVER]",
  "address": { "city": "string", "state": "string", "pincode": "string" }
}
```

## 4. Admission / Encounter (`Aivana.Encounter`)
Represents a specific physical stay at the hospital.
```json
{
  "encounterId": "UUID",
  "patientId": "UUID",
  "hospitalId": "UUID",
  "type": "enum[INPATIENT, DAYCARE, OUTPATIENT, EMERGENCY]",
  "admissionDate": "ISO8601",
  "dischargeDate": "ISO8601",
  "treatingDoctorId": "UUID"
}
```

## 5. Diagnosis (`Aivana.Diagnosis`)
Represents a clinical condition evaluated during the Encounter.
```json
{
  "diagnosisId": "UUID",
  "encounterId": "UUID",
  "icd10Code": "string",
  "description": "string",
  "type": "enum[PRIMARY, SECONDARY, COMORBIDITY]",
  "isPresentOnAdmission": "boolean"
}
```

## 6. Procedure (`Aivana.Procedure`)
Represents a surgical or medical intervention.
```json
{
  "procedureId": "UUID",
  "encounterId": "UUID",
  "cptCode": "string",
  "description": "string",
  "datePerformed": "ISO8601",
  "surgeonId": "UUID"
}
```

## 7. Claim (`Aivana.Claim`)
Represents the financial demand sent to the TPA.
```json
{
  "claimId": "UUID",
  "encounterId": "UUID",
  "policyId": "UUID",
  "claimType": "enum[PRE_AUTH, ENHANCEMENT, FINAL_BILL, POST_HOSPITALIZATION]",
  "totalClaimedAmount": "number",
  "lineItems": [
    {
      "itemId": "UUID",
      "category": "enum[ROOM_RENT, SURGEON_FEE, PHARMACY, INVESTIGATION]",
      "description": "string",
      "amount": "number"
    }
  ],
  "status": "enum[DRAFT, READY, SUBMITTED, PENDING_QUERY, APPROVED, DENIED, SETTLED]"
}
```

## 8. Evidence (`Aivana.Evidence`)
Anchors a clinical fact to a physical document (Generated by EGS).
```json
{
  "evidenceId": "UUID",
  "factId": "UUID (e.g. DiagnosisId)",
  "document": {
    "documentId": "UUID",
    "name": "string",
    "pageNumber": "integer"
  },
  "boundingBox": { "x1": 0, "y1": 0, "x2": 100, "y2": 100 },
  "confidenceScore": "number"
}
```

## 9. Denial (`Aivana.Denial`)
Represents the TPA's rejection or deduction of a claim.
```json
{
  "denialId": "UUID",
  "claimId": "UUID",
  "type": "enum[FULL_DENIAL, PARTIAL_DEDUCTION]",
  "totalDeductedAmount": "number",
  "reasons": [
    {
      "code": "string",
      "description": "string",
      "deductedAmount": "number"
    }
  ]
}
```

## 10. Appeal (`Aivana.Appeal`)
Represents Aivana's counter-argument (Generated by Aegis).
```json
{
  "appealId": "UUID",
  "denialId": "UUID",
  "status": "enum[DRAFT, SUBMITTED, WON, LOST]",
  "generatedDraftText": "string",
  "evidenceReferences": ["EvidenceId_1", "EvidenceId_2"]
}
```

## 11. Settlement (`Aivana.Settlement`)
Represents the final financial reconciliation.
```json
{
  "settlementId": "UUID",
  "claimId": "UUID",
  "approvedAmount": "number",
  "patientShareAmount": "number",
  "utrNumber": "string",
  "settledDate": "ISO8601"
}
```

## 12. Rule (`Aivana.Rule`)
Represents a deterministic condition from AKS.
```json
{
  "ruleId": "UUID",
  "packId": "UUID",
  "category": "enum[CLINICAL, FINANCIAL, ADMINISTRATIVE]",
  "conditionLogic": "string (DSL)",
  "action": "enum[DEDUCT, REJECT, FLAG_FOR_REVIEW]"
}
```

## 13. Audit (`Aivana.Audit`)
Represents a system action for compliance.
```json
{
  "auditId": "UUID",
  "actor": { "type": "enum[SYSTEM, HUMAN]", "id": "string" },
  "action": "enum[VIEW, CREATE, UPDATE, DELETE, OVERRIDE]",
  "resourceType": "string",
  "resourceId": "string",
  "timestamp": "ISO8601"
}
```

## 14. Notification (`Aivana.Notification`)
Represents a communication sent to a stakeholder.
```json
{
  "notificationId": "UUID",
  "channel": "enum[SMS, EMAIL, WHATSAPP, IN_APP]",
  "recipient": "string",
  "templateId": "string",
  "status": "enum[SENT, DELIVERED, FAILED]"
}
```
