# Enterprise Error Catalog

To prevent generic "500 Internal Server Error" responses from cascading through the system and causing debugging nightmares, Aivana enforces a strict Enterprise Error Catalog. Every microservice must emit structured errors using these specific codes.

---

## Error Envelope Schema
All errors thrown internally or externally must match this JSON structure:
```json
{
  "errorCode": "INS-4012",
  "message": "Room rent cap exceeded by 5%.",
  "serviceId": "TAIGA_FINANCIAL",
  "traceId": "gw-88192-abc",
  "severity": "WARN",
  "actionable": true,
  "context": {
    "policyId": "POL-99",
    "calculatedCap": 5000,
    "actualCharge": 5250
  }
}
```

---

## INS-1000: Ingestion & OCR Errors
Generated by the Docling Ingestion Gateway and ID services.
* **INS-1001**: `UNSUPPORTED_FILE_FORMAT` (E.g., Hospital uploaded a `.doc` instead of `.pdf`).
* **INS-1002**: `FILE_CORRUPTED` (PDF cannot be parsed).
* **INS-1003**: `OCR_RESOLUTION_TOO_LOW` (DPI under 150, image illegible).
* **INS-1004**: `DOCUMENT_CLASS_UNKNOWN` (AI cannot identify if this is a lab report or bill).

## INS-2000: Clinical Intelligence Errors (Fairway)
Generated by Fairway and TPR.
* **INS-2001**: `MISSING_CLINICAL_EVIDENCE` (Rule requires ECG, but no ECG document exists).
* **INS-2002**: `CONTRADICTORY_EVIDENCE` (Admission says Diabetic, Discharge says Non-Diabetic. Requires TPR resolution).
* **INS-2003**: `INVALID_ICD_MAPPING` (The extracted diagnosis text cannot be confidently mapped to an ICD-10 code).

## INS-3000: Policy & Rule Errors (AKS/PKG)
Generated by the Knowledge Studio or Policy Graph.
* **INS-3001**: `RULE_PACK_NOT_FOUND` (Hospital config references Taiga v1.5, but v1.5 was deprecated).
* **INS-3002**: `CIRCULAR_RULE_DEPENDENCY` (A rule in AKS relies on a rule that relies on itself).
* **INS-3003**: `GRAPH_TRAVERSAL_TIMEOUT` (Evidence Graph query took > 5 seconds).

## INS-4000: Financial Compliance Errors (Taiga)
Generated by the Taiga Financial Engine.
* **INS-4001**: `ROOM_RENT_CAP_VIOLATION` (Patient in ICU, but policy only allows General Ward).
* **INS-4002**: `UNBUNDLING_DETECTED` (Hospital billed for surgery and anesthesia separately when they should be packaged).
* **INS-4003**: `CONSUMABLE_NON_PAYABLE` (Billed for gloves/masks which are excluded by IRDAI norms).
* **INS-4004**: `MATH_RECONCILIATION_FAILED` (Line items sum to ₹50k, but the total bill says ₹55k).

## INS-5000: Submission & Orchestration Errors (MCO/FCP)
Generated by the MCO or Submission Adapters.
* **INS-5001**: `CLAIM_LOCKED_MUTATION_ATTEMPT` (Hospital tried to add a document to a claim that was already submitted).
* **INS-5002**: `SAGA_TIMEOUT` (A human review step took longer than the SLA of 48 hours).
* **INS-5003**: `TPA_PORTAL_UNREACHABLE` (The external Insurer's API is returning 503).
* **INS-5004**: `FCP_VALIDATION_FAILED` (The generated claim packet is missing mandatory TPA fields).

## INS-6000: Integration & Protocol Errors (Hub)
Generated by the Integration Hub.
* **INS-6001**: `HL7_PARSE_ERROR` (Invalid MLLP framing or missing mandatory segments).
* **INS-6002**: `UNAUTHORIZED_HOSPITAL_IP` (Traffic received from an IP not on the hospital's VPN whitelist).
* **INS-6003**: `MAPPING_SCRIPT_EXCEPTION` (The V8 Javascript adapter threw a runtime error).

## INS-7000: AI & Model Errors (AI Gateway)
Generated by the AI Model Gateway.
* **INS-7001**: `LLM_PROVIDER_OUTAGE` (All primary and fallback models are timing out).
* **INS-7002**: `JSON_SCHEMA_VIOLATION` (The LLM returned output that failed Zod validation).
* **INS-7003**: `HALLUCINATION_DETECTED` (The Guardrails engine flagged the output as mathematically or logically impossible).
* **INS-7004**: `PROMPT_NOT_FOUND` (Requested `aegis-v2`, but it does not exist in the Registry).

## INS-8000: Security & Auth Errors (IAM)
Generated by the API Gateway or IAM service.
* **INS-8001**: `JWT_EXPIRED` (User session ended).
* **INS-8002**: `ABAC_VIOLATION` (Doctor from Branch A tried to view Claim from Branch B).
* **INS-8003**: `RATE_LIMIT_EXCEEDED` (Hospital made > 1000 requests per minute).
* **INS-8004**: `PII_LEAK_PREVENTED` (Guardrails caught a raw patient name attempting to exit the VPC to OpenAI).
